### 1.2 与 lab-report-agent 的差异

| 维度 | lab-report-agent | 本项目 |
|------|-----------------|--------|
| 框架 | Next.js 全栈 | React + Vite (前端) + Python FastAPI (后端) |
| 输入 | 1个 DOCX 模板 | PDF 资料 + Word 模板（两个文件） |
| 编辑方式 | 用户自然语言指令 | Agent 自动填写 + 用户微调 |
| 文档引擎 | SuperDoc SDK (Node.js) | SuperDoc SDK (Node.js) + Python 桥接 |
| AI 工具 | SuperDoc 内置 4 个工具 | 同，通过 Node.js 子进程调用 |

## 四、API 设计

### 4.1 PDF 解析

```
POST /api/pdf/parse
Content-Type: multipart/form-data

Body: file (PDF 文件)

Response: SSE 流式
- event: progress  → { percent: 50, message: "解析第5页..." }
- event: result    → { text: "...", pages: 20, tables: [...] }
- event: done
```

### 4.2 创建会话

```
POST /api/session/create
Content-Type: multipart/form-data

Body: 
- template (DOCX 文件)
- bid_data (JSON 字符串，PDF 解析结果)

Response: JSON
{
  "sessionId": "uuid",
  "revision": 1,
  "docUrl": "/api/session/{sessionId}/docx?rev=1"
}
```

### 4.3 Agent 填写

```
PUT /api/session/create
Content-Type: application/json

Body:
{
  "sessionId": "uuid",
  "message": "请根据PDF资料填写模板中的所有待填位置",
  "mode": "auto"  // auto | manual
}

Response: SSE 流式
- event: start
- event: status    → { message: "模型推理中（第1轮）..." }
- event: turn_start → { round: 1 }
- event: thought   → { content: "正在分析模板结构..." }
- event: token     → { content: "填" }
- event: tool_call_start → { name: "superdoc_edit", args: {...} }
- event: tool_call_end   → { result: "..." }
- event: doc_checkpoint  → { docUrl: "/api/session/{id}/docx?rev=2" }
- event: done      → { reply: "填写完成", docUrl: "..." }
- event: error     → { message: "..." }
```

### 4.4 获取文档

```
GET /api/session/{sessionId}/docx?rev={revision}

Response: DOCX 文件流
```

### 4.5 取消 Agent

```
POST /api/session/create/cancel
Content-Type: application/json

Body: { "sessionId": "uuid" }

Response: { "ok": true }
```

## 六、Agent 循环详细设计

### 6.1 系统提示词

```python
SYSTEM_PROMPT = """
你是一个投标文件填写助手。你的任务是根据招标文件（PDF）的资料，填写投标模板（Word）中的待填位置。

## 规则
1. 格式绝对不能破坏 — 只修改内容，不修改格式
2. 不编造 — PDF 中没有的信息标注【待人工补充】
3. ★条款必须逐一响应 — 零遗漏
4. 填写后自查 — 确保准确性、完整性、格式完整性

## 可用工具
- superdoc_get_content: 读取文档内容
- superdoc_edit: 插入/替换内容
- superdoc_format: 格式化文本
- superdoc_mutations: 批量操作

## 当前文档信息
{doc_info}

## PDF 资料
{bid_data}

## 公司信息
{company_info}
"""
```

### 6.2 ReAct 循环

```python
async def run_agent_loop(session, llm, bid_data, company_info):
    messages = [
        {"role": "system", "content": build_system_prompt(bid_data, company_info)},
        {"role": "user", "content": "请填写模板中的所有待填位置"},
    ]
    
    for round in range(MAX_ROUNDS):  # 30轮
        # 1. 调用 LLM
        response = await llm.chat(messages, tools=SUPERDOC_TOOLS)
        
        # 2. 如果没有 tool_calls，结束
        if not response.get("tool_calls"):
            break
        
        # 3. 执行每个 tool_call
        for tool_call in response["tool_calls"]:
            result = await dispatch_tool(session, tool_call)
            messages.append({"role": "tool", "content": result})
            
            # 4. 保存文档 + 发送 checkpoint
            session.save()
            yield {"event": "doc_checkpoint", "docUrl": session.get_url()}
        
        # 5. 自查
        issues = await check_completeness(session, bid_data)
        if not issues:
            break
```

## 七、SSE 事件类型

| 事件 | 说明 | 数据 |
|------|------|------|
| `start` | 流开始 | `{}` |
| `status` | 状态消息 | `{ message: "模型推理中..." }` |
| `turn_start` | 新一轮开始 | `{ round: 1 }` |
| `plan_created` | 执行计划创建 | `{ steps: [...] }` |
| `plan_step_started` | 计划步骤开始 | `{ step: "填写封面" }` |
| `plan_step_finished` | 计划步骤完成 | `{ step: "填写封面" }` |
| `thought` | LLM 思考过程 | `{ content: "..." }` |
| `token` | 流式文本 | `{ content: "填" }` |
| `tool_call_start` | 工具调用开始 | `{ name: "superdoc_edit", args: {...} }` |
| `tool_call_end` | 工具调用结束 | `{ result: "..." }` |
| `doc_checkpoint` | 文档保存检查点 | `{ docUrl: "..." }` |
| `done` | 完成 | `{ reply: "...", docUrl: "..." }` |
| `error` | 错误 | `{ message: "..." }` |
